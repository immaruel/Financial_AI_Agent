🧭

Query Planner

str → QueryPlan

🕸️

Graph Retriever

QueryPlan → SubGraphResult

Seed 탐색, hop/edge 전략으로 관련 서브그래프 추출

📎

Evidence Retriever

SubGraphResult → EvidenceResult

PassageIndex 기반 원문 근거 회수 + 상충 탐지

🧠

Causal Reasoner

Subgraph + Evidence → CausalChain

시간순 정렬, 인과 연결, unsupported inference 점검

✅

Hypothesis Checker

CausalChain → Verdict

unsupported claim, contradiction, 시간 역전 검증

📝

Answer Composer

All Results → Structured Answer

사실/해석 구분, citation alignment, confidence 산출

🛡️

StructuredAnswer → Final Answer

FAILURE TAXONOMY

금융 특화 오류 분류 체계

금융 AI가 만들 수 있는 8가지 실패 유형을 사전 정의하고, 각 단계에서 감지합니다

F1 엔티티 혼동

현대차 ≠ 현대모비스
계열사/본사 혼동, ticker mismatch

F2 시간축 오류

발표일 ≠ 효력일
최신 정보 대신 과거 정보 사용

F3 숫자 오류

절대값 ≠ 증감률
금액/비율/단위 오류

F4 이벤트 귀속

주체 기업 ≠ 대상 기업
행위자와 영향받는 기업 뒤바뀜

F5 근거 누락

핵심 citation 부족
반대 evidence 누락

F6 추론 과장

상관관계 → 인과 단정
가능성을 사실처럼 표현

F7 안전성 오류

투자 권유, 과도한 확신
리스크 경고 누락

F8 과잉 방어

답할 수 있는 질문도 회피
유용성 훼손

QUERY TYPES

4가지 질의 유형

Q: "현대차에 최근 어떤 이벤트가 있었는가?"
Query Planner → Company(현대차) seed 탐색 → 연결된 Event 노드 수집 → 시간순 정렬 + Evidence 반환
특정 기업 또는 산업에 일어난 사실 기반 질의. KG에서 직접 조회 가능한 정보를 반환합니다.
Q: "이 이벤트가 어떤 기업군에 전이될 가능성이 있는가?"
Event seed → INVOLVES / SUPPLIES_TO Edge Traversal → Multi-Hop Sub-graph → 관계 기반 추론
엔티티 간 구조적 관계를 다단계로 탐색. 공급망, 경쟁구도, 정책 수혜/피해 관계를 파악합니다.
Q: "어떤 이벤트가 먼저 발생했고, 이후 무엇이 연쇄적으로 이어졌는가?"
PRECEDES Edge Traversal → CausalChain Reconstruction → 타임라인 정렬 → 패턴 비교
이벤트의 시간 연쇄와 패턴 추적. 과거 유사 이벤트와 비교하여 전개 방향을 분석합니다.
Q: "지금 시장에서 핵심 촉발 요인은 무엇인가?"
전체 KG 스캔 → 높은 빈도 Event 클러스터 → 영향 범위 산출 → 우선순위 랭킹
복합 분석이 필요한 고수준 질의. 여러 에이전트가 협업하여 의사결정 근거를 구성합니다.

WHY

왜 만들었는가

01

일반 RAG로 금융 질문을 던져보았습니다

기업 혼동, 시간 오류, 근거 없는 인과 추론 — 금융 도메인에서 텍스트 기반 RAG는 예상보다 자주 틀렸고, 왜 틀렸는지 추적할 방법이 없었습니다.

02

지식 그래프 + 하네스 엔지니어링을 조합했습니다

텍스트를 구조화된 지식으로 변환하고, 에이전트의 행동을 Constraint·Context·Verification·Feedback Loop 4계층으로 감시하는 시스템을 설계했습니다.

03

AI가 틀리는 것보다, 왜 틀렸는지 모르는 것이 더 위험합니다

이 시스템의 핵심은 "틀리지 않는 AI"가 아닙니다. 실패를 감지하고, 분류하고, 스스로 복구하거나, 적절한 시점에 인간에게 넘기는 것입니다.
